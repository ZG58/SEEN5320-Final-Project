\begin{abstract}
This project builds a closed optimization workflow for a pressure swing adsorption (PSA) CO$_2$ capture process using only the existing scalar simulation records in \texttt{data/manifest.csv}. Deleted time-space CSV files are not used. A manifest-only surrogate maps seven PSA operating variables to purity, recovery, productivity, and specific energy. The surrogate is then coupled with a Python implementation of NSGA-II to solve two bi-objective problems: purity versus recovery, and productivity versus energy. The optimized inputs and surrogate-predicted outputs are exported for detailed-model reruns through CSV files, completing the data handoff needed for closed-loop comparison.
\end{abstract}

\section{Objective and Workflow}
The objective is to replace deleted-file-dependent analysis with a practical surrogate-assisted optimization workflow. The research route is:
\begin{enumerate}
    \item Train surrogate models from the 1000 successful rows in \texttt{manifest.csv}.
    \item Use the surrogate inside a multi-objective genetic algorithm.
    \item Export optimized decision variables to the detailed PSA model.
    \item Compare detailed-model outputs against surrogate predictions after the detailed reruns are available.
\end{enumerate}

The workflow is intentionally manifest-only. Auxiliary file-path columns in the manifest are ignored, and only scalar design variables and KPI outputs are read.

\section{Surrogate Model}
The surrogate uses a degree-3 polynomial response surface in standardized design variables. A fixed 80/20 train-test split with random seed 42 is used. Ridge regularization is selected by five-fold cross-validation on the training set.

The regression targets are CO$_2$ recovery, productivity, and $\log_{10}$(specific energy). Purity is modeled as $\log_{10}(p - 0.15)$, where $p$ is CO$_2$ purity and 0.15 is the feed CO$_2$ fraction. This transformation improves the fit near feed-level purity. During optimization, predictions are clipped to the observed manifest output envelope to avoid reporting extrapolated KPI values outside the training data range.

\section{Genetic Algorithm Optimization}
The surrogate is optimized with an in-house NumPy implementation of NSGA-II. The GA operates directly on the seven bounded design variables and uses the surrogate for fast KPI evaluation. The settings are:
\begin{itemize}
    \item Population size: 120.
    \item Generations: 200.
    \item Simulated binary crossover probability: 0.9.
    \item Polynomial mutation probability: $1/7$ per variable.
    \item Initial population: half manifest-seeded points and half random points inside the configured bounds.
\end{itemize}

Two bi-objective optimization problems are solved:
\[
\max_x \left[purity(x), recovery(x)\right],
\]
and
\[
\max_x \left[productivity(x), -\log_{10}(energy(x))\right].
\]
The final nondominated set from each problem contains 120 surrogate Pareto candidates. Twenty representative candidates from each front are exported to \path{data/detailed_model_input.csv}.

\section{Optimization Results}
For the purity-recovery problem, the surrogate Pareto set covers predicted purity from 0.436 to 0.670 and recovery from 0.913 to 1.000. The corresponding productivity remains low, around 5.5--5.9 mol kg$^{-1}$ h$^{-1}$, showing the expected tradeoff between separation quality and throughput.

\section{Closed-Loop Detailed-Model Interface}
The closed-loop handoff is file based. The GA writes:
\begin{itemize}
    \item \path{data/ga_optimization_all.csv}: final GA populations for both problems.
    \item \path{data/ga_optimization_nondominated.csv}: final nondominated candidates.
    \item \path{data/detailed_model_input.csv}: 40 representative candidates for detailed PSA reruns.
    \item \path{data/detailed_model_results_template.csv}: the expected detailed-model result format.
\end{itemize}
After detailed simulations are completed, their results should be saved as \path{data/detailed_model_results.csv}. The comparison script will then generate \path{data/detailed_model_comparison.csv} and error summaries.

\section{Conclusions}
The project now follows a closed, manifest-only route: surrogate training, GA-based multi-objective optimization, candidate export, and detailed-model comparison. The surrogate is strongest for recovery, productivity, and log energy. Purity is harder to learn from the scalar manifest alone, so optimized purity candidates should be treated as screening points that require detailed-model confirmation. The two GA fronts reveal different operating regimes: purity-recovery optimization favors low-throughput enriched cases, while productivity-energy optimization favors feed-level purity with much higher throughput.
