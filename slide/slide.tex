\begin{frame}{Project Question}
\begin{block}{Research question}
Can machine learning turn a PSA simulation campaign into accurate surrogates and interpretable operating guidance?
\end{block}
\begin{itemize}
    \item System: four-step/extended-cycle PSA CO$_2$/N$_2$ separation with Zeolite 13X.
    \item Data: 1000 successful LHS samples and final-cycle time-space profiles.
    \item ML tasks: regression, high-performance classification, PCA/K-means regime discovery.
    \item Goal: reduce simulator dependence while preserving physical interpretation.
\end{itemize}
\end{frame}

\begin{frame}{From Profiles to ML Features}
\begin{columns}[T,onlytextwidth]
\begin{column}{0.50\textwidth}
\begin{itemize}
    \item Each CSV stores the last returned cycle over time and bed position.
    \item For each PSA step, features summarize means, ranges, final inlet/outlet values, axial gradients, loading selectivity proxies, and gas-wall temperature lag.
    \item These features connect scalar KPIs to the internal state of the bed.
\end{itemize}

\begin{frame}{Machine Learning Workflow}
\begin{columns}[T,onlytextwidth]
\begin{column}{0.48\textwidth}
\textbf{Supervised learning}
\begin{itemize}
    \item Regression targets: purity, recovery, productivity, $\log_{10}$(energy).
    \item Models: Ridge, random forest, gradient boosting, SVR, MLP.
    \item Validation: fixed 80/20 train-test split.
\end{itemize}
\end{column}
\begin{column}{0.48\textwidth}
\textbf{Screening and interpretation}
\begin{itemize}
    \item Classify top 20\% balanced-score samples.
    \item Compute four-objective Pareto set.
    \item Use PCA/K-means to identify operating regimes.
\end{itemize}
\end{column}
\end{columns}
\end{frame}

\begin{frame}{Conclusions}
\begin{itemize}
    \item ML surrogates can reproduce PSA simulation outputs accurately from one completed LHS campaign.
    \item Final-cycle profile features are valuable because they expose internal bed states that design variables alone miss.
    \item The sampled design space shows a severe purity-recovery-energy conflict.
    \item Practical next step: expand the domain with material variables and optimize around purity constraints, not only balanced KPI score.
\end{itemize}
